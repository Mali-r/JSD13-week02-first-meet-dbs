## 1. Describe the human problems
ฝั่งลูกค้า :
 - ใช้เวลาทำนาน ไม่อยากเสียเวลาทำเอง
 - หาซื้อแซลมอนดอง/กุ้งดองที่สด สะอาด ราคาชัดเจนยาก — ร้านส่วนใหญ่ไม่มีเมนูออนไลน์ให้ดูราคา/ขนาดก่อนตัดสินใจ ต้องทักถามทีละร้าน
 - ไม่รู้ว่าน้ำหนัก/ไซซ์ที่จะซื้อ (100g/250g/500g/1kg) ราคาต่างกันเท่าไหร่ ต้องรอถามหน้าร้าน
 - ลูกค้าประจำไม่มีระบบสมาชิก/ส่วนลดสะสม ทำให้ไม่รู้สึกคุ้มค่าที่จะกลับมาซื้อซ้ำ
 - ไม่มีประวัติการสั่งซื้อเก่าให้ดูย้อนหลัง (ซื้ออะไรไปบ้าง ราคาเท่าไหร่)

ฝั่งเจ้าของร้าน/พนักงาน :
 - จดออเดอร์และคิดเงินด้วยมือ/กระดาษ ผิดพลาดง่าย  เสียเวลาหน้าร้านตอนคนแน่น
 - ไม่มีระบบสต็อกที่อัปเดตอัตโนมัติ ทำให้ขายสินค้าหมดสต็อกโดยไม่รู้ตัว หรือสั่งวัตถุดิบเพิ่มไม่ทัน
 - ไม่มีระบบเก็บ "ใคร (พนักงาน) ทำรายการไหน" และไม่มีการจำกัดสิทธิ์การเข้าถึงข้อมูล — พนักงานทุกคนแก้ราคา/ลบสินค้าได้เหมือนกันหมด ซึ่งเสี่ยงต่อความผิดพลาด/ทุจริต
 - เจ้าของร้านไม่มีเครื่องมือดูรายงานยอดขายภาพรวม ต้องรวมยอดเองจากใบเสร็จ

## 2. Describe How your solution solves these problems
ปัญหา : วิธีแก้
1. ลูกค้าไม่เห็นราคา/ไซซ์ก่อนซื้อ : หน้า Menu แสดงสินค้าพร้อมรูป ราคา และตัวเลือกไซซ์ (100g/250g/500g/1kg) ให้ดูก่อนตัดสินใจ
2. ไม่มีระบบสมาชิก/ส่วนลด : ผูกออเดอร์กับ สมาชิก (member) ได้ที่หน้า POS สะสมประวัติการซื้อ
3. ไม่มีประวัติการซื้อ : เก็บทุกออเดอร์ใน MongoDB (orders) ดูย้อนหลังได้จากหน้า History 
4. คิดเงินหน้าร้านช้า/ผิดพลาด : หน้า POS คิดราคา/ตะกร้า/ยอดรวมอัตโนมัติ ลดความผิดพลาดจากการคิดเลขมือ
5. สต็อกไม่อัปเดต : ทุกออเดอร์ที่ปิดการขายตัดสต็อกสินค้าอัตโนมัติจาก products.stock
6. ไม่มีการจำกัดสิทธิ์พนักงาน : เพิ่มหน้า User Management ให้ admin เพิ่ม/ลบ/แก้ไขผู้ใช้ และกำหนด role (admin / staff / customer) แยกสิทธิ์การเข้าถึงชัดเจน
7. ไม่มีรายงานยอดขายภาพรวม : หน้า รายงานยอดขาย สรุปยอดขายจากข้อมูล orders ทั้งหมดในระบบ
8. ไม่รู้กำไรที่เเท้จริง : ต้องมี BOM เพราะต้นทุนหมุนอยู่ตลอด

## 3. Describe your ecommerce business idea including core features
ไอเดียธุรกิจ: ร้านขายแซลมอนดองและกุ้งดองพร้อมทาน แบบ hybrid — ขายหน้าร้าน (POS)
ควบคู่กับเว็บเมนูออนไลน์ให้ลูกค้าดูสินค้า/ราคาก่อนมาซื้อหรือสั่งล่วงหน้า
เจาะกลุ่มลูกค้าที่ชอบอาหารทะเลดองสไตล์เกาหลี/ญี่ปุ่น ที่อยากได้ของสดสะอาด ราคาชัดเจน
และร้านที่อยากมีระบบจัดการหลังบ้านที่เป็นระเบียบ
Core features :
1. Menu (หน้าเมนูสินค้า) — แสดงแซลมอนดอง/กุ้งดอง พร้อมรูป ราคา ไซซ์ (100g/250g/500g/1kg), หมวดหมู่ (salmon/shrimp/sauce/side)
2. Cart & Checkout (POS) — เพิ่มสินค้าลงตะกร้า, เลือกสมาชิก, คิดเงิน, พิมพ์/แสดงใบเสร็จ
3. Member system — ผูกลูกค้าเป็นสมาชิก เก็บประวัติ/แต้มสะสม/รายการโปรด
4. Product management — admin เพิ่ม/แก้ไข/ลบสินค้า, ปรับสต็อก, เปิด-ปิดการขาย
5. User management (ใหม่) — admin เพิ่ม/แก้ไข/ลบผู้ใช้ระบบ พร้อมกำหนด role (admin/staff/customer)
6. Order history & Sales report — ดูประวัติออเดอร์รายบุคคล และรายงานยอดขายรวมสำหรับเจ้าของร้าน
7. Database (MongoDB) — เก็บข้อมูล 3 collection หลัก: users, products, orders (ตาม schema ที่ออกแบบไว้)
8. BOM — เจ้าของสามารถรู้กำไรที่เเท้จริงได้
